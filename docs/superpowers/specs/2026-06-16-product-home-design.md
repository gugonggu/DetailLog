# Product Home Design

## Direction

Detailog의 `/`는 단순 MVP 랜딩이 아니라 실제 서비스의 공개 홈으로 보이게 만든다. 랜딩을 삭제해 `/community`로 바로 보내지는 않는다. 대신 제품 소개와 공개 세차 기록 미리보기를 함께 배치하는 하이브리드 홈을 사용한다.

## Scope

- `/`에서 브랜드 메시지, 주요 CTA, 최근 공개 세차 기록 미리보기 최대 3개를 보여준다.
- 공개 미리보기는 로그인하지 않은 방문자도 볼 수 있어야 한다.
- 전체 커뮤니티, 좋아요, 북마크, 작성 등 앱 기능은 기존 인증 영역에 둔다.
- 공개 데이터 노출은 `visibility = 'public'` 세차 기록, 해당 이미지, 공개 프로필 뷰의 닉네임/아바타로 제한한다.

## UX

첫 화면은 `Detailog`를 명확히 보여주고 “세차 기록이 루틴이 되는 공간”이라는 제품 가치를 전달한다. 동시에 실제 공개 기록 카드를 노출해 서비스가 비어 보이지 않게 한다. 로그인 사용자는 대시보드로, 신규 방문자는 회원가입/커뮤니티 둘러보기로 이동할 수 있다.

## Data

마케팅 홈은 Supabase anon client로 공개 세차 기록을 읽는다. 이를 위해 별도 마이그레이션에서 anon에게 public wash log read 정책을 추가한다. 비공개 기록은 RLS 조건과 앱 쿼리 조건 양쪽에서 제외한다.

## Testing

커뮤니티 서비스에 랜딩 미리보기 매핑 테스트를 추가한다. 전체 테스트, lint, build, 브라우저 렌더링으로 검증한다.
